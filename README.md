# 📢 Бот-Заклик для Telegram груп

## Встановлення

### 1. Отримай токен бота
- Напиши [@BotFather](https://t.me/BotFather) у Telegram
- Команда `/newbot`, задай ім'я та юзернейм
- Скопіюй токен (виглядає як `123456789:ABC-DEF...`)
- **Важливо:** у BotFather вимкни Privacy Mode (`/setprivacy → Disable`), щоб бот бачив всі повідомлення в групі

### 2. Запуск локально (Python 3.11+)

```bash
pip install -r requirements.txt
export BOT_TOKEN="твій_токен_тут"
python bot.py
```

### 3. Запуск через Docker

```bash
docker build -t zaklyk-bot .

docker run -d \
  --name zaklyk-bot \
  --restart unless-stopped \
  -e BOT_TOKEN="твій_токен_тут" \
  -v zaklyk_data:/data \
  zaklyk-bot
```

### 4. Деплой на Railway

1. Додай `Procfile` у корінь:
   ```
   worker: python bot.py
   ```
2. У Railway Dashboard → Variables додай: `BOT_TOKEN=твій_токен`
3. Підключи GitHub репозиторій або використай Railway CLI:
   ```bash
   railway login && railway init && railway up
   ```
> ⚠️ SQLite скидається при редеплої на Railway. Для збереження даних підключи Railway Volume і встанови `DB_PATH=/data/bot.db`.

## Команди

### 📢 Заклик
| Команда | Дія |
|---------|-----|
| `!заклик`, `!скликати`, `/call` | Скликати всіх учасників (кулдаун 1 хв) |
| `/fix_call` | Перевірити стан заклику (тільки власник чату) |

Можна додати коментар: `!скликати хто ще не задонатив?`

### ✅ Реєстрація в закликі
| Команда | Дія |
|---------|-----|
| `!заклик+`, `заклик+`, `!call+`, `!прокинувся`, `!прокинулась`, `!з хлібом`, `/reg` | Повернутись до заклику в цьому чаті |
| `!заклик++`, `!call++`, `/reg_all` | Повернутись до заклику в усіх чатах |

### ❌ Вихід із заклику
| Команда | Дія |
|---------|-----|
| `!заклик-`, `заклик-`, `!call-`, `!оф`, `!піти по хліб`, `!тиха година`, `!я сплю`, `/unreg` | Вийти з заклику в цьому чаті |
| `!заклик--`, `/unreg_all` | Вийти з заклику в усіх чатах |

### 🔍 Додатково
| Команда | Дія |
|---------|-----|
| `!заклик?` | Перевірити свій статус та смайлик (або статус іншого — у відповідь на повідомлення) |
| `!смайлик 🔥`, `/emoji 🔥`, `/set_emoji 🔥` | Змінити свій смайлик у закликах |
| `!перепис населення`, `/regchat` | Запустити реєстрацію всіх учасників (тільки власник чату) |

### 📊 Статистика
| Команда | Дія |
|---------|-----|
| `!моя стата`, `/mystats` | Особиста статистика: кількість повідомлень за день / тиждень / місяць / рік / весь час + дата першого повідомлення |
| `!стата`, `/stats` | Топ-10 учасників за сьогодні + загальна кількість за день |
| `!стата тиждень`, `!тижнева стата`, `/stats_week` | Топ-10 учасників за останні 7 днів + загальна кількість за тиждень |
| `!вся стата`, `/stats_all` | Топ-10 учасників за весь час + загальна кількість |

### ⚙️ Налаштування (тільки власник чату)
| Команда | Дія |
|---------|-----|
| `/call_del <сек>` | Автовидалення закликів через N секунд (0 — вимкнути) |
| `/call_admins_only` | Дозволити заклик тільки адмінам (перемикач вкл/викл) |

### 📥 Імпорт історії чату
| Команда | Дія |
|---------|-----|
| `/import_chat` | Інструкція як імпортувати експорт з Telegram Desktop |
| `/import_chat <chat_id>` (+ файл) | Імпортувати `result.json` — наповнює базу учасниками та статистикою повідомлень |

**Як імпортувати:**
1. У Telegram Desktop відкрий потрібний чат
2. `⋮ → Експорт історії чату → JSON` (без медіафайлів)
3. Надішли файл `result.json` боту **в приватний чат**
4. У підписі до файлу або окремим повідомленням напиши:
   `/import_chat -1001234567890` (ID групи)
5. ID групи можна дізнатись через [@username_to_id_bot](https://t.me/username_to_id_bot)

> **Ліміти імпорту через бота:** Telegram Desktop не обмежує кількість повідомлень в експорті — вся доступна тобі як учаснику/адміну історія буде вивантажена. Файл може важити від кількох КБ до сотень МБ залежно від активності чату. Бот приймає файли до **20 МБ** (стандартний ліміт Telegram Bot API). Якщо файл більший — використовуй прямий імпорт нижче.

### ⚡ Прямий імпорт (без ліміту розміру)

Якщо `result.json` більший за 20 МБ або ти хочеш імпортувати напряму на сервері — використовуй скрипт `import_history.py`.

**Локально** (бот і база на твоєму комп'ютері):
```bash
python import_history.py result.json -1001234567890
```

**На Railway:**
1. Відкрий Railway Dashboard → твій сервіс → вкладка **Shell**
2. Завантаж файл у репозиторій тимчасово (або скористайся `curl`/`scp`)
3. Виконай у Shell:
   ```bash
   python import_history.py result.json -1001234567890
   ```
4. Після імпорту видали файл із репо

Повторний запуск скрипту безпечний — дублів не буде (`INSERT OR IGNORE`). Файл будь-якого розміру обробляється напряму з диска.

## Як бот збирає статистику

- Кожне повідомлення в групі автоматично логується в таблицю `message_log`
- Статистика накопичується з моменту додавання бота до чату
- Для повної історії (до додавання бота) — використовуй імпорт через `/import_chat`
- Учасники автоматично реєструються в закликах при першому повідомленні

## Структура бази даних (SQLite)

```
members        — учасники закликів (chat_id, user_id, emoji, active)
message_log    — лог повідомлень для статистики (chat_id, user_id, sent_at)
chat_settings  — налаштування чатів (delete_after, admins_only)
hidden_users   — службова таблиця
```

База зберігається у файлі `bot.db` (або `/data/bot.db` в Docker/Railway).
